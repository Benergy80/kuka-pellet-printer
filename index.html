The Core Idea

The robot does only three things to the extruder

⚡

RUN / STOP


Digital output $OUT[n]

Extrude on print moves, stop on travel moves.

🔄

SET SPEED


Analog output $ANOUT[1]

0–10 V proportional to sliced flow rate.

⚠️

READ FAULT


Digital input $IN[n]

Pause the print on an extruder fault.

✅

Everything thermal regulates itself.

Barrel heat zones, the heated bed, the chamber, and the dryer each run on their own controllers. The robot never manages temperature — which keeps the integration small and robust.

Platform — the three things still to build

Build platform & enclosure

▦ 1 · Rigid print table, tied to the robot's base frame ▶
  1. Size the build area to the reach — keep it ~1.0–1.5 m square, set 0.7–1.5 m in front of the base.
  2. Build a stiff steel weldment with a thick flat top plate (≥15–20 mm) — it must not flex when the heavy arm decelerates.
  3. Anchor it to the floor, ideally the same slab as the robot, so they can't move relative to each other.
  4. Add 3 hard reference features (dowels / machined pockets) at known XY — these become the base-calibration points.
  5. Mount the heated bed on top, on a thermal break so the frame doesn't sink the heat.
Why it matters: origin + 2 points teach the print coordinate system, so slice Z=0 equals the real table top. If the table shifts, Z accuracy is lost.
🌡️ 2 · Heated build bed — adhesion & warp control ▶
  1. Build surface: flat aluminum tooling plate or borosilicate glass, sized to the build area.
  2. Heat with a high-wattage 240V silicone pad bonded underneath (LFAM beds run ~1.5 kW to 5 kW+).
  3. Control as an independent loop — thermocouple → PID controller → SSR → heater.
  4. Mount on the table over insulation + thermal-break standoffs to protect it and save energy.
  5. Adhesion: PEI sheet or glue, plus a wide brim/raft in the slice for big beads.
Control loop: Thermocouple → PID controller → solid-state relay → silicone heater. Runs by itself — the robot never sets bed temperature.
🛡️ 3 · Enclosure, fencing & safety interlocks ▶
  1. Machine guarding (required): fence beyond the swept volume, one interlocked door, E-stops inside and out. A KR 150/180 at speed is lethal.
  2. Thermal & fume chamber: insulated / poly panels keep a warm chamber (~60 °C, up to 150 °C) and contain fumes — duct an extractor out.
  3. Wire the door interlock into Operator Safety (X11 pins C8–C10 / C9–C11, dual channel).
  4. Wire external E-stops to X11 pins C14–C15 / C16–D1.
  5. Replace the factory bridges with the real safety devices — never leave them jumpered.
⚠️

Safety-circuit wiring must be done and validated by a qualified machine-safety integrator — not improvised with permanent jumpers.

Electrical & I/O

The DeviceNet I/O island

The KR C2 talks to the world over DeviceNet. Add a small remote-I/O island; that is the whole integration core.

KR C2
DeviceNet master
→
BK5120
DeviceNet coupler · MACID 11
→
KL4004 — Analog Out
$ANOUT[1] → extruder speed (0–10 V)
KL2xxx — Digital Out
$OUT[n] → extruder RUN / heater enable
KL1xxx — Digital In
$IN[n] ← extruder ready / fault

Mapped in iosys.ini

ANOUT1=11,0,16,3 outb0=11,8,x3 ; full-scale 0x6C00 = 27648 → 10 V

Calibrate so $ANOUT[1]=1.0 → 10 V = max extrude rate. Bench-test before any print.

Power

Licensed electrician. 480V 3-phase to the KR C2, sized to its nameplate (~7.5–13 kVA — not the sum of axis drive ratings). The heated bed/chamber, extruder control box, dryer, and autoloader each go on their own circuits.
⚠️

Confirm the extruder control box accepts an external RUN + speed input before ordering the I/O island.

Software

From model to robot motion

1
CAD → Cura

Slice the STL in Ultimaker Cura with the CuraToKRC2 plugin.

2
Configure

krl_config.json: tool A/B/C, HOME, Tool & Base numbers, flow constant. Enable Relative Extrusion.

3
Export KRL

Native .src/.dat, auto-split into subroutines to fit KR C2 RAM.

4
Transfer

Copy files to the controller via USB or network share.

5
Run

Robot follows the path; writes $ANOUT[1] for flow, toggles $OUT[n] for extrude on/off.

Upgrade paths: KUKA|prc (Rhino/Grasshopper) for simulation & non-planar toolpaths · RoboDK + KUKA.RSI for live streaming (~4–12 ms) — add later, no drive changes.
Calibration

Tie the model, the nozzle, and the table together

🎯

Tool (TCP) — XYZ 4-point

Setup ▸ Measure ▸ Tool ▸ XYZ 4-point

Touch the nozzle tip to one fixed point from 4 orientations; the controller computes the TCP. Enter the tool A/B/C to match krl_config.json. Save as the Tool number.

📐

Base — 3-point

Setup ▸ Measure ▸ Base ▸ 3-point

With the taught TCP, touch the table's 3 reference features: origin, a point on +X, a point in the XY plane. This builds the print coordinate frame. Save as the Base number.

✅

Verify

Jog to Base 0,0,0 — the nozzle should sit at the table origin; +X and +Y run along the table edges. Re-touch the references after any crash.
